Getting Started with Consul in Docker

Every node need
$ PUBLIC_IP=”$(ifconfig eno16777984| grep inet | grep netmask | awk ‘{ print $2}’)”

two other node need to setting
$ JOIN_IP=192.168.235.101

$ $(docker run –rm progrium/consul cmd:run $PUBLIC_IP -it)
$ $(docker run –rm progrium/consul cmd:run $PUBLIC_IP::$JOIN_IP -it)

You can open the web ui on and consul ip port 8500
192.168.183.101:8500/ui/

consul dns need to change docker config

Setting
$ sudo vim /etc/systemd/system/docker.service
Append on ExecStart
–dns 172.17.42.1 –dns 8.8.8.8 –dns-search service.consul
$ sudo systemctl daemon-reload
$ sudo systemctl restart docker

Setting host dns to be the same with docker
$ sudo vim /etc/systemd/resolved.conf
Add DNS=172.17.42.1 140.113.235.1 8.8.8.8
$ sudo systemctl restart systemd-resolved

lookup node
$ dig @172.17.42.1 coreos1.node.consul
lookup service
$ dig @172.17.42.1 consul.service.consul

consul node name base on hostname
consul service name base on image name

Using registrator with consul and docker

Start registrator
$ docker run -d \
–name=registrator \
–net=host \
-h $HOSTNAME \
–volume=/var/run/docker.sock:/tmp/docker.sock \
gliderlabs/registrator:latest \
consul://$PUBLIC_IP:8500

And run some docker container
$ docker run -d -P redis

consul service name base on image name
$ dig @172.17.42.1 redis.service.consul
And you will get the machine ip
;; ANSWER SECTION:
redis.service.consul. 0 IN A 172.17.0.5

$ dig @172.17.42.1 -t SRV redis.service.consul
And you will get the machine ip and mapping port 32769
;; ANSWER SECTION:
redis.service.consul. 0 IN SRV 1 1 32769 coreos1.node.dc1.consul.
You can acess the redis by setting coreos1.node.dc1.consul:32769

environment SERVICE_NAME will rewrite the default image name
$ docker run -d -P -e “SERVICE_NAME=db” redis
environment SERVICE_TAGS will rewrite the consul tag
$ docker run -d -P -e “SERVICE_NAME=db” -e “SERVICE_TAGS=primary” redis

Then you can dns search you custom SERVICE_NAME
$ dig @172.17.42.1 db.service.consul
$ dig @172.17.42.1 -t SRV db.service.consul

Also can search your SERVICE_TAGS
$ dig @172.17.42.1 -t SRV primary.db.service.consul

You can custom you dockerfile to this tag
dockerfile
ENV SERVICE_NAME db

$ dig db.service.consul SRV +short

Using consul to discover docker swarm node

$ docker run –rm -p 2375:2375 swarm join –advertise= consul:///
$ docker rum –rm -p 2375:2375 swarm manage -H tcp:// consul:///
Example.
$ docker run –rm -p 2375:2375 swarm join –advertise=192.168.235.101:2375 consul://192.168.235.101/nodes
$ docker run –rm -p 2375:2375 swarm join –advertise=192.168.235.102:2375 consul://192.168.235.101/nodes
$ docker run –rm -p 2375:2375 swarm join –advertise=192.168.235.103:2375 consul://192.168.235.101/nodes

$ docker run –rm -p 4000:2375 swarm manage consul://192.168.235.101/nodes

$ docker -H tcp://192.168.235.101:4000 info

$ docker run –rm swarm join –addr=192.168.235.101:2375 consul://192.168.235.101:8500/swarm
$ docker run -d swarm join –addr=192.168.235.102:2375 consul://192.168.235.101:8500/swarm
$ docker run -d swarm join –addr=192.168.235.103:2375 consul://192.168.235.101:8500/swarm

$ docker run -d -p 6666:2375 swarm manage consul://192.168.235.101:8500/swarm
